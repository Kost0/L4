# cut-CLI

Распределенная система для обработки текстовых файлов с функциональностью утилиты `cut`. Проект реализует архитектуру Master-Worker для параллельной обработки данных.

## Описание

cut-CLI — это распределенная система, которая позволяет обрабатывать текстовые файлы, извлекая из них определенные поля или символы, аналогично UNIX-утилите `cut`. Система состоит из мастер-узла, который распределяет задачи, и воркеров, которые выполняют обработку данных.

## Структура проекта

```
cut-CLI/
├── cmd/
│   ├── master/    # Главный узел для распределения задач
│   └── worker/    # Рабочие узлы для обработки данных
├── internal/      # Внутренняя логика приложения
├── config.yaml    # Конфигурационный файл
├── benchmark.sh   # Скрипт для тестирования производительности
└── test*.txt      # Тестовые файлы
```

## Использование

### 1. Запуск воркеров

Для корректной работы системы необходимо запустить несколько воркеров. Каждый воркер запускается в отдельном терминале. Используйте флаг -p для указания адреса в формате host:port:  

```bash
# Воркер 1 (запустится по адресу localhost:8081)
go run cmd/worker/main.go

# Воркер 2 (в новом терминале)
go run cmd/worker/main.go -p localhost:8082

# Воркер 3 (в новом терминале)
go run cmd/worker/main.go -p localhost:8083
```

Рекомендуется запускать минимум 2-3 воркера для эффективной распределенной обработки.

### 2. Запуск мастер-узла

Перед запуском в файле config.yaml следует указать адреса всех воркеров.

```bash
go run cmd/master/main.go
```


## Тестирование

Для проверки производительности системы используйте скрипт `benchmark.sh`:

```bash
chmod +x benchmark.sh
./benchmark.sh
```

Скрипт выполнит тест и покажет метрики производительности распределенной обработки.
